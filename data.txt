┌──────────────────────────────────────────────────────┐
│  STEP 1 — EVENT ORIGIN                               │
│  Failed login occurs on client                       │
│  Payload: { userId, ipAddress, timestamp,            │
│             userAgent, region, appId }               │
└──────────────────────┬───────────────────────────────┘
                       │
                       ▼
┌──────────────────────────────────────────────────────┐
│  STEP 2 — INGESTION API                              │
│  POST /api/events/failed-login                       │
│                                                      │
│  ┌── Validate payload ──────────────────────────┐   │
│  │  • Required fields present?                  │   │
│  │  • IP format valid?                          │   │
│  │  • Timestamp not in future?                  │   │
│  └──────────────┬───────────────────────────────┘   │
│                 │                                    │
│    ┌─── NO ─────┘                                    │
│    │                                                 │
│    ▼                                                 │
│  Return 400 Bad Request                              │
│    │                                                 │
│    └─── YES ──► Normalize & enrich                  │
│                 • Assign eventId (GUID)              │
│                 • UTC-normalize timestamp            │
│                 • Lookup geo-region if missing       │
│                 • Push to Redis Stream / RabbitMQ   │
│                 • Return 202 Accepted               │
└──────────────────────┬───────────────────────────────┘
                       │
                       ▼
┌──────────────────────────────────────────────────────┐
│  STEP 3 — EVENT QUEUE (Redis Stream)                 │
│  Key: login_events_stream                            │
│  Consumer group: processors                          │
│  Message retained for 24 h (configurable)            │
└──────────────────────┬───────────────────────────────┘
                       │  Worker polls or listens
                       ▼
┌──────────────────────────────────────────────────────┐
│  STEP 4 — PROCESSING WORKER                          │
│                                                      │
│  A. INSERT into login_events (PostgreSQL)            │
│                                                      │
│  B. Update Redis sliding windows                     │
│     • INCR  user:{userId}:count   EX 300  (5 min)   │
│     • INCR  ip:{ip}:count         EX 300             │
│     • SADD  ip:{ip}:users  userId EX 300             │
│     • ZADD  spike:global  ts  eventId               │
│                                                      │
│  C. Call Detection Engine with event context         │
└──────────────────────┬───────────────────────────────┘
                       │
                       ▼
┌──────────────────────────────────────────────────────┐
│  STEP 5 — DETECTION ENGINE (rule evaluation loop)   │
│                                                      │
│  ┌─ Rule 1: Brute Force ──────────────────────────┐ │
│  │  IF user:{userId}:count > BRUTE_FORCE_THRESHOLD │ │
│  │  THEN → fire BRUTE_FORCE alert                 │ │
│  └────────────────────────────────────────────────┘ │
│                                                      │
│  ┌─ Rule 2: IP Sweep ─────────────────────────────┐ │
│  │  IF SCARD ip:{ip}:users > IP_SWEEP_THRESHOLD   │ │
│  │  THEN → fire IP_SWEEP alert                    │ │
│  └────────────────────────────────────────────────┘ │
│                                                      │
│  ┌─ Rule 3: Distributed Attack ───────────────────┐ │
│  │  query DB: distinct IPs for userId in 10 min   │ │
│  │  IF count > DISTRIBUTED_THRESHOLD              │ │
│  │  THEN → fire DISTRIBUTED alert                 │ │
│  └────────────────────────────────────────────────┘ │
│                                                      │
│  ┌─ Rule 4: Global Spike ─────────────────────────┐ │
│  │  ZCOUNT spike:global (now-60s) now             │ │
│  │  IF count > SPIKE_THRESHOLD                    │ │
│  │  THEN → fire SPIKE alert                       │ │
│  └────────────────────────────────────────────────┘ │
│                                                      │
│  Any rules fired?                                    │
│    NO  ──► ACK queue message, loop back             │
│    YES ──► Pass alert(s) to Alert Module            │
└──────────────────────┬───────────────────────────────┘
                       │ (one or more alerts)
                       ▼
┌──────────────────────────────────────────────────────┐
│  STEP 6 — ALERT MODULE                               │
│                                                      │
│  A. DEDUPLICATION                                    │
│     • Key: alertType + userId/ip + 5-min window     │
│     • Identical alert already active?               │
│         YES → increment count, skip re-insert       │
│         NO  → continue                              │
│                                                      │
│  B. SEVERITY SCORING                                 │
│     score = normalize(attemptCount, uniqueIPs,       │
│                        windowDuration)               │
│     score 0–40  → LOW                               │
│     score 41–70 → MEDIUM                            │
│     score 71–100→ HIGH / CRITICAL                   │
│                                                      │
│  C. AI SUMMARY (optional, async)                    │
│     Call LLM API with raw stats                     │
│     → "User admin had 42 failed logins from 6       │
│         IPs in 3 minutes. Possible brute force."    │
│                                                      │
│  D. PERSIST → alerts table (PostgreSQL)             │
│                                                      │
│  E. NOTIFY                                           │
│     • SignalR → Angular dashboard (< 1 s)           │
│     • Email / Slack (severity HIGH+)                │
└──────────────────────┬───────────────────────────────┘
                       │
                       ▼
